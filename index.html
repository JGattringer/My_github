Guia de Comandos Git

Uma referência completa dos principais comandos Git para desenvolvedores. Este guia contém os comandos mais utilizados no dia a dia, organizados por categoria para facilitar a consulta.

Comandos Básicos

Inicializar repositório

git init

Inicializa um novo repositório Git no diretório atual, criando a pasta .git com todos os arquivos necessários para o controle de versão.

Clonar repositório

git clone <url>

Cria uma cópia local de um repositório remoto, baixando todos os arquivos e histórico de commits para sua máquina.

Verificar status

git status

Mostra o estado atual do repositório, incluindo arquivos modificados, adicionados ou não rastreados pelo Git.

Adicionar arquivos

git add <arquivo>

Adiciona arquivos específicos à área de staging (preparação) para serem incluídos no próximo commit.

Adicionar todos os arquivos

git add .

Adiciona todos os arquivos modificados e novos do diretório atual à área de staging de uma só vez.

Fazer commit

git commit -m "mensagem"

Salva as alterações da área de staging no histórico do repositório com uma mensagem descritiva das mudanças realizadas.

Ver histórico

git log

Exibe o histórico de commits do repositório, mostrando informações como autor, data e mensagem de cada commit.

Ver diferenças

git diff

Mostra as diferenças entre os arquivos no diretório de trabalho e a última versão commitada.

Branches

Listar branches

git branch

Lista todas as branches locais do repositório, destacando com asterisco (*) a branch atual em que você está trabalhando.

Criar nova branch

git branch <nome>

Cria uma nova branch com o nome especificado, baseada na branch atual, sem mudar para ela automaticamente.

Mudar de branch

git checkout <branch>

Muda para a branch especificada, atualizando os arquivos do diretório de trabalho para refletir o estado dessa branch.

Criar e mudar para branch

git checkout -b <nome>

Cria uma nova branch e muda para ela automaticamente em um único comando, economizando tempo e passos.

Mudar de branch (moderno)

git switch <branch>

Comando mais moderno para mudar entre branches, introduzido para separar as funcionalidades do checkout tradicional.

Criar e mudar (moderno)

git switch -c <nome>

Versão moderna do comando para criar uma nova branch e mudar para ela, equivalente ao checkout -b.

Fazer merge

git merge <branch>

Integra as alterações da branch especificada na branch atual, combinando os históricos de desenvolvimento.

Deletar branch

git branch -d <nome>

Remove uma branch local que já foi mergeada, liberando espaço e mantendo o repositório organizado.

Colaboração

Adicionar repositório remoto

git remote add origin <url>

Adiciona um repositório remoto com o nome "origin", estabelecendo a conexão com um servidor Git como GitHub ou GitLab.

Enviar alterações

git push origin <branch>

Envia os commits locais da branch especificada para o repositório remoto, compartilhando suas alterações com outros desenvolvedores.

Configurar upstream

git push -u origin <branch>

Envia a branch e configura o tracking automático, permitindo usar apenas "git push" nos próximos envios.

Baixar alterações

git pull origin <branch>

Baixa e integra as alterações do repositório remoto na branch local, mantendo seu código atualizado com o time.

Baixar sem integrar

git fetch

Baixa as alterações do repositório remoto sem integrá-las automaticamente, permitindo revisar antes de fazer merge.

Ver repositórios remotos

git remote -v

Lista todos os repositórios remotos configurados, mostrando as URLs para fetch e push de cada um.

Configurar usuário

git config --global user.name "Nome"

Define o nome do usuário que aparecerá nos commits, configuração essencial para identificar o autor das alterações.

Configurar email

git config --global user.email "email@exemplo.com"

Define o email do usuário para os commits, importante para identificação e notificações em plataformas como GitHub.

Comandos Avançados

Reverter commit

git revert <commit>

Cria um novo commit que desfaz as alterações de um commit específico, mantendo o histórico limpo e rastreável.

Reset suave

git reset --soft HEAD~1

Desfaz o último commit mantendo as alterações na área de staging, permitindo refazer o commit com modificações.

Reset misto

git reset HEAD~1

Desfaz o último commit e remove as alterações da área de staging, mantendo as modificações no diretório de trabalho.

Stash alterações

git stash

Salva temporariamente as alterações não commitadas, permitindo mudar de branch ou fazer pull sem perder o trabalho atual.

Aplicar stash

git stash pop

Restaura as alterações salvas mais recentemente pelo stash e remove essa entrada da pilha de stashes.

Rebase interativo

git rebase -i HEAD~n

Permite reorganizar, editar ou combinar os últimos n commits, criando um histórico mais limpo e organizado.

Cherry-pick

git cherry-pick <commit>

Aplica as alterações de um commit específico de outra branch na branch atual, sem fazer merge completo.

Buscar no histórico

git log --grep="texto"

Busca commits que contenham o texto especificado na mensagem, útil para encontrar alterações específicas no histórico.
